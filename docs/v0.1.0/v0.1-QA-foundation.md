# v0.1 Vocabulary Module - QA Checklist

## Foundation

### Configuration
- [ ] Config file created at `~/.config/langtut/config.yaml` on first run
- [ ] Config directory created with 0755 permissions
- [ ] Config file created with 0600 permissions
- [ ] Default config returned when file doesn't exist
- [ ] Config loads existing YAML correctly
- [ ] Config saves changes correctly
- [ ] Config handles missing `presets` field (initializes empty map)
- [ ] Config handles invalid YAML (returns error)
- [ ] Config handles missing `api_key` field
- [ ] Config handles missing `target_language` field (defaults to empty)
- [ ] API key prompting when missing
- [ ] API key saved after prompting

### API Client (Gemini)
- [ ] Client creation with valid API key
- [ ] Client creation fails with empty API key
- [ ] Client creation fails with invalid API key format
- [ ] `GenerateWordInfo` returns meaning and examples
- [ ] `GenerateWordInfo` handles API timeout (30s default)
- [ ] `GenerateWordInfo` handles network errors
- [ ] `GenerateWordInfo` handles API errors (invalid key, rate limit)
- [ ] Response parsing handles markdown format correctly
- [ ] Response parsing handles missing sections (fallback to plain text)
- [ ] Response parsing handles malformed markdown
- [ ] Response parsing extracts numbered list examples
- [ ] Response parsing extracts bullet list examples
- [ ] Response parsing handles continuation lines in examples
- [ ] Response parsing handles empty meaning (defaults to "No meaning generated")
- [ ] Response parsing handles empty examples (defaults to ["No examples generated"])
- [ ] `StreamChat` handles `iterator.Done` correctly (not treated as error)
- [ ] `StreamChat` handles timeout correctly
- [ ] `StreamChat` handles context cancellation
- [ ] Client closes resources correctly

### Preset System
- [ ] Default presets available (eli5, intermediate, advanced)
- [ ] `GetPreset` returns default (intermediate) for empty name
- [ ] `GetPreset` returns preset by name (case-insensitive)
- [ ] `GetPreset` returns error for unknown preset
- [ ] `ListPresets` returns all default presets

## Integration

### Command Line
- [ ] `langtut vocab` defaults to list view
- [ ] `langtut vocab list` opens list view
- [ ] `langtut vocab add <word>` opens add modal
- [ ] `langtut vocab add` without args shows error
- [ ] `langtut vocab add word1 word2 word3` processes all
- [ ] Invalid subcommand shows help

### Config Integration
- [ ] API key loaded from config
- [ ] Target language loaded from config (defaults to "English")
- [ ] Config created if missing
- [ ] Config errors handled gracefully

### Library Persistence
- [ ] Words persist across sessions
- [ ] Metadata updated on save (`LastUpdated` timestamp)
- [ ] Library version maintained
- [ ] Words map initialized if nil
- [ ] Library reloaded after external changes (not currently, but should be tested)

### Error Handling
- [ ] All errors wrapped with context
- [ ] Errors displayed in TUI (red text)

- [ ] Errors don't crash application
- [ ] Graceful degradation on non-critical errors

## Known Limitations (Not Bugs)

- Library not auto-reloaded on external file changes
- No concurrent API request handling
- Examples limited to 3 in UI (more can be stored)
- No undo/redo functionality
- No bulk operations
- No import/export yet
